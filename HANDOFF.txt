    • RECOGNIZE flashcards. ORDER ALGORITHM per row of 5: phase forward (A→O, in order) until all 5
      graded "Know it" → phase reverse (O→A) until all 5 "Know it" → phase random (weighted across all
      introduced). Header shows phase. localStorage 'fkti_kana' per script {rows,phase,pk,cards:{lvl,streak,seen}}.

    • QUIZ GATE: unlocking the next row requires a 5-question multiple-choice quiz (kana→romaji),
      perfect score to pass (buildGate/renderGate/addRow).
    • WRITE: finger-tracing over a faint guide char on canvas (Clear / Hide-guide / Next).
 ✅ ✏️ Stroke order — KanjiVG. ▶ Watch animates strokes in order/direction (~2.2s/stroke, slow).
    Draw it → Check grades each stroke for START POINT + DIRECTION + COUNT (catches シ vs ツ).
    Detection: parse SVG <path> per stroke, sample via getPointAtLength on an offscreen measure-svg,
    normalize to 0-1, compare start-dist (<0.26) + direction dot (>0.4). Prev/Next, per script.
 ✅ 🔊 Sounds & Extras — teaching + tap-to-check practicelets: dakuten ゛/handakuten ゜(voiced+bilabial p),
    yōon combos (small ゃゅょ), long vowels (おう/ー), small っ (sokuon).
 ✅ 📊 Progress & decks — GOJŪON CHART color-coded by mastery (green=Know it, yellow=Kinda, orange=Don't
    really, red=No clue, grey=Not started); tap a cell to hear it. RESPONSIVE: portrait = rows by
    consonant; LANDSCAPE/DESKTOP (matchMedia orientation:landscape | min-width:680) = TRADITIONAL vertical
    gojūon (vowels あいうえお down the rightmost column, consonant columns right→left; placed by vowelRow()),
    re-renders on resize/orientationchange. Drives 4 CUSTOM DECKS by confidence band (bandPool):
    💪 boost (lvl3), 🙂 little (≥lvl2), 😤 challenge (≥lvl1), 🔥 hurt (everything); deck grades feed the chart.

SUMO AT NAGOYA — sumo.html (built 2026-06-29 for Greg's class trip to the Nagoya basho, Day 8 / nakabi,
  Sun 19 Jul 2026, IG Arena). Single self-contained file, offline-first, same Deeply-Connected theme.
  Home menu -> sections (‹ back nav), like kana.html. Six sections:
   • Today's Matchups — Day-8 torikumi (Makuuchi/Juryo). • Banzuke — full ranking, tap any wrestler -> bio.
   • Sumo Primer • Ranks & the Banzuke • Ringside Glossary • The 82 Techniques (kimarite).
  LIVE DATA: sumo-api.com (free, CORS open). BASHO='202607', NAKABI_DAY=8. getJSON() caches each response
   to localStorage and falls back to it offline (graceful "not published yet" states until the JSA releases
   the banzuke ~late June and each day's torikumi the evening before). Bios: /rikishi/:id + /:id/stats.
  PHOTOS (wrestlers): pluggable slot in avaSrc() -> CARICATURE[id] (drop-in override) -> PHOTOMAP (Wikimedia
   CC, else JSA) -> stylized placeholder. Map built by sumo-src/photos.js. Re-run after the final banzuke.
  KIMARITE: 82 techniques + 7 non-techniques in KIM[]. Each card = a PHOTO (📷, 22 PD bout photos, duotoned)
   if one exists in KPHOTO, else an ORIGINAL FKTI physics diagram from kdiagram(arch). Searchable/filterable.
  THE DIAGRAMS (kdiagram): articulated crouched rikishi (rik(): 2-segment tapered limbs, mawashi, ōichō
   topknot; o.y = belly centre; feet/hands absolute coords; a()=attacker terracotta face+1, d()=defender slate
   face-1). Each technique maps to an 'arch' (pose, drawn per real grip/footwork) AND a MECH (physics family).
   Overlay shows WHY it works: ● center of gravity, shaded base of support, red force/rotation, ▲ fulcrum,
   ✕ removed base, teal=momentum, + red rings (solid=hold/grip, dashed=push/strike). Legend + physics primer
   in-page. WHY[mech] = one-line explanation on every card. 11 mechanisms; ~27 arch poses.

autonomous maintenance bot (budget ~$10/mo; he's fine ~$3-5). It keeps the FKTI apps fresh & accurate by
doing ONE tiny "beat" per scheduled run.

LIVES IN: caretaker/ + .github/workflows/caretaker.yml (in this repo).
 - caretaker/ledger.json .... rotating checklist of "watches" (id, active, cadenceDays, lastChecked,
     instructions). Each beat picks the oldest DUE active watch. Active: `sumo-gossip`; `vegan-dining`
     (DISCOVERY→REVIEW — finds veg spots for the trip cities via the LIVE fkti-dining-app data at
     https://fkti-dining-app.pages.dev/data/<city>.json so it doesn't duplicate, stages them in

 - caretaker/BEAT.md ........ the brief the agent runs each beat: pick oldest due watch → verify via web →
     apply FACTUAL updates + cite sources; SENSITIVE items (allegations/rumors about named people) go to
     review-queue, NEVER auto-published; append 1 line to changelog; bump lastChecked; STOP. One watch/run.
 - caretaker/changelog.md ... the visible "what changed" log.
 - caretaker/review-queue.md  human-review holding pen for sensitive finds (nothing here is live).
 - .github/workflows/caretaker.yml  daily cron (+ manual "run a beat" button). Installs Claude Code,
